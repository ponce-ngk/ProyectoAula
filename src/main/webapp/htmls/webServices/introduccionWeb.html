Introduccion a la web services

Un web service o servicio web es un software basado en texto que funciona con Internet. Este sistema tiene la función de permitir la transmisión de solicitudes y respuestas entre diferentes servidores o aplicaciones, independientemente de las diferencias que existan entre los lenguajes de programación en los que fueron desarrollados o la plataforma sobre la que se ejecutan[1].

En otras palabras, un servicio web es, como sugiere su nombre, un servicio que permite la comunicación de máquina a máquina y el intercambio de datos entre aplicaciones en una red de Internet[1].

No se puede mencionar el término "servicios web" hoy sin mencionar inmediatamente referencias como Web Services (AWS) o el servicio web de Google. Estos gigantes tecnológicos se llevaron a casa el trofeo al abordar la necesidad de desarrollar aplicaciones. La escala de empresas como Amazon y Google es exactamente lo que hace posibles los servicios web modernos[1].

Uno de los elementos clave de este sistema es XML (Extensible Markup Language o Lenguaje de Marcado Extensible en español). Es la forma en que se codifican o programan los datos para que puedan ser procesados por diferentes sistemas, es decir, es un lenguaje estándar que permite la comunicación incluso cuando las aplicaciones o sistemas utilizan diferentes lenguajes de programación[2].

Arquitectura de servicios web

• Descubrimiento del servicio: Responsable de centralizar los servicios web en un directorio común de revistas y proporcionar una funcionalidad de búsqueda y publicación sencilla. UDDI maneja el descubrimiento de servicios[3].

• Descripción del servicio: Uno de los aspectos más distintivos de los servicios web es que se describen a sí mismos. Esto significa que una vez que se ha identificado un servicio web, nos proporcionará información sobre las operaciones que admite y cómo habilitarlo. Esto se hace a través del lenguaje de descripción de servicios web (WSDL)[3].

• Invocación de servicios: Llamar a un servicio web implica pasar mensajes entre el cliente y el servidor. El Protocolo simple de acceso a objetos (SOAP) especifica cómo formatear los mensajes de solicitud al servidor y cómo el servidor debe formatear sus mensajes de respuesta[3].

• Transportación: Todos estos mensajes deben transmitirse de alguna manera entre el servidor y el cliente. El protocolo elegido para ello es HTTP (Protocolo de transferencia de hipertexto). Se pueden utilizar otros protocolos, pero HTTP es actualmente el más utilizado[3].

Estructura de un servicio web

Lenguaje de descripción de servicios web (WSDL): este es el archivo XML que describe el servicio web. La solicitud de servicio web utiliza este archivo para enlazar con el servicio[4].

SOAP: este es el protocolo basado en XML utilizado por la solicitud de servicio web para invocar el servicio[4].

Para obtener un escenario más detallado, consulte la información de descripción general del escenario de servicios web para obtener más información sobre la historia de un minorista en línea ficticio de suministros de jardinería llamado Plants by WebSphere y cómo ese minorista adoptó el concepto de servicios web[4].
