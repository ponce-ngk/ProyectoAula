¿Que es una API?

Una interfaz de programa de aplicación (API) define las reglas que se deben seguir para comunicarse con otros sistemas de software. Los desarrolladores exponen o crean API para que otras aplicaciones puedan comunicarse con sus aplicaciones mediante programación. Por ejemplo, la aplicación Timesheet expone una API que requiere el nombre completo y el intervalo de fechas de un empleado. Después de recibir esta información, procesa internamente la hoja de horas del empleado y devuelve la cantidad de horas trabajadas en ese rango de fechas. Se puede pensar en una API web como una puerta de enlace entre los clientes y los recursos en la web[1].

Una API es un conjunto de funcionalidades o recursos que nos expone un sistema para poder interactuar con él desde otro sistema, independientemente de los lenguajes de programación en los que esté escrito cada uno de ellos o de la tecnología u otra, y que gracias a ella trabaja con un nivel de abstracción[2].

Podemos ver un ejemplo para entender mejor el concepto, la API de un banco, que nos expondrá sus servicios, digamos para hacer una compra online, así de esta forma desde nuestra aplicación, por ejemplo, una App Android puede llenar un rol en nuestra cuenta bancaria usando la API que nos proporciona el propio banco, sin darnos el código ni los detalles técnicos de cómo hacerlo, porque no nos importa[2].

¿Qué es la API REST?

Una API RESTful es un estilo arquitectónico para una interfaz de programa de aplicación (API) que usa solicitudes HTTP para acceder y usar datos. Estos datos se pueden usar para los tipos de datos GET, PUT, POST y DELETE, que se refieren a operaciones de lectura, actualización, creación y eliminación relacionadas con los recursos[3].

Su acrónimo deriva de Representational State Transfer, es un estilo de arquitectura de software para la comunicación cliente-servidor. Se basa en el protocolo HTTP para comunicarse con el servidor, y los mensajes enviados y recibidos pueden estar en formato XML o JSON, de los que hablaremos más adelante[2].

La tecnología REST generalmente se prefiere a otras tecnologías similares. Esto suele ocurrir porque REST utiliza menos ancho de banda, lo que lo hace más adecuado para un uso eficiente de Internet. Las API RESTful también se pueden crear con lenguajes de programación como JavaScript o Python[3].

¿Cómo funciona una API RESTFULL?

REST determina la estructura de una API. Los desarrolladores están sujetos a un conjunto específico de reglas cuando diseñan una API. Por ejemplo, una ley establece que un enlace a una URL debe devolver cierta información. Cada URL se denomina solicitud, mientras que los datos devueltos se denominan respuesta[4].

La API REST desglosa una transacción para generar una secuencia de pequeños componentes. Cada componente se ocupa de un aspecto fundamental específico de una transacción. Este modularidad lo convierte en un enfoque de desarrollo flexible. Una API REST aprovecha los métodos HTTP descritos por el protocolo RFC 2616[4].

Utiliza las siguientes solicitudes HTTP: solicitudes de búsqueda de datos, solicitudes PUT para cambiar el estado de los datos (como un objeto, archivo o bloque), solicitud POST para crear datos, eliminar la solicitud para eliminarla

¿Cuáles son los beneficios de las API RESTful?

Escalabilidad: Los sistemas que implementan API REST pueden escalar de manera eficiente porque REST optimiza las interacciones cliente-servidor. La tecnología sin estado elimina la carga del servidor porque el servidor no necesita retener información de solicitudes de clientes anteriores. El almacenamiento en caché bien administrado elimina parcial o completamente algunas interacciones cliente-servidor. Todas estas funciones admiten la escalabilidad sin causar cuellos de botella en la comunicación que reduzcan el rendimiento[1].

Flexibilidad: Los servicios web RESTful admiten la separación completa entre el cliente y el servidor. Simplifican y desacoplan los diversos componentes del servidor para que cada parte pueda evolucionar de forma independiente. Los cambios de plataforma o tecnología realizados en la aplicación del servidor no afectan a la aplicación del cliente. La capacidad de superponer la funcionalidad de la aplicación aumenta aún más la flexibilidad. Por ejemplo, los desarrolladores pueden realizar cambios en la capa de la base de datos sin tener que volver a escribir la lógica de la aplicación[1].

Independencia: Las API REST son independientes de la tecnología. Puede escribir aplicaciones del lado del cliente y del lado del servidor en múltiples lenguajes de programación sin afectar el diseño de la API. También puede cambiar la tecnología subyacente en ambos lados sin afectar la comunicación[1].
