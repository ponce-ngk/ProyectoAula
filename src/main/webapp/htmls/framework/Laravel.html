Introducción Laravel

Definición

Laravel es el framework PHP más conveniente para principiantes y programadores avanzados. Acelere el desarrollo de aplicaciones web y llévelas al mercado utilizando métodos PHP modernos orientados a objetos. Su expresiva sintaxis y sus características modernas atraerán a los desarrolladores que buscan crear aplicaciones [1].

Los marcos facilitan el proceso de desarrollo al conectar diferentes módulos entre sí. El marco web de proporciona un punto de partida para crear aplicaciones, lo que permite a los desarrolladores de concentrarse más [1].

Además, incluye algunas características de tecnologías como ASP.NET, MVC, CodeIgniter, Ruby on Rails, etc [1].

Por lo tanto, todas sus características acelerarán el desarrollo web. Si conoce los conceptos básicos de PHP y algunas secuencias de comandos PHP avanzadas, Laravel te hará la vida más fácil [1].

Laravel sigue el patrón de arquitectura modelo-vista-controlador (MVC) [2].


MVC

MVC es una arquitectura de software... que separa la lógica de dominio/aplicación/negocio del resto de la interfaz de usuario hace esto separando la aplicación en tres partes: modelo, vista y controlador [2].

El modelo administra el comportamiento y gestión de datos porque puede satisfacer solicitudes de información proporcionando instrucciones para cambiar su estado de información; y notifique a los observadores en el sistema de eventos cuando cambie la información. puede ser una base de datos cualquier número de estructuras de datos o sistemas de almacenamiento. Eso es por así decirlo, es gestión de aplicaciones y datos [2].

Las vistas proporcionan efectivamente los elementos de la interfaz de usuario de la aplicación. Muestra os datos del modelo en un formato adecuado para la interfaz de usuario [2].

Como la mayoría de los frameworks PHP, Laravel también tiene una estructura basada en los estándares MVC. Es un modelo arquitectónico que divide la aplicación en 3 capas:

• M de model: Representa la capa de la base de datos [2].

• V de view: Representa la capa de presentación, visible para el usuario [2].

• C de controller: Representa una regla de negocio, básicamente la lógica que tendrá su aplicación con la base de datos y la información enviada por el usuario [2].

Además de la arquitectura MVC, Laravel hace que el desarrollo de aplicaciones sea simple y rápido. Ofrece muchas soluciones modernas que son fáciles de configurar y se adaptan a los problemas que enfrentan todos los desarrolladores web durante la creación, que incluyen:

• Ruta [2].

• Autentificación [2].

• Permiso [2].

• Control de secciones [2].

• Manipulación de la base de datos [2].

• Validación [2].


Enrutamiento

La función principal de un marco de aplicación web es recibir solicitudes de usuarios y enviar respuestas, generalmente a través de HTTP (S). Entonces, la primera tarea en el proceso de registro es decidir qué ruta desea. Sin rutas , el sistema no puede contactarlo. Básicamente, una ruta es una URL que le permite comunicarse con el mundo exterior utilizando una URL conocida [4].

Básicamente, el cliente crea una solicitud para una ruta específica y la pasa al controlador para que maneje la solicitud según sea necesario. La carpeta /routes en la raíz del proyecto contiene los archivos de rutas [1].

El enrutamiento es necesario cuando se construyen sistemas de información utilizando el framework Laravel [3].

El enrutamiento es el proceso por el cual los artículos pueden llegar a su destino de una ubicación a otra. Para el marco de Laravel, el elemento en cuestión es el sitio web de la página. En Laravel, las rutas de aplicaciones web están en el archivo route.php dentro del directorio de la aplicación [3].

Algunos ejemplos de enrutamiento en el archivo route/ web.php

Caso1:

Route::get('ruta1', function () { return view('bienvenido'); });

En este ejemplo, 'Ruta' tiene un método 'get' que devuelve una vista de 'Bienvenida' que muestra una página web [4].

Cuando el usuario visita la página de inicio, se le redirige a la página de bienvenida. Esta vista de "Bienvenida" es en realidad un archivo PHP: "welcome.blade.php" en el directorio resources/views [4].

Caso2:

Route::post('route2', 'App\Http\Controllers\FileController@read');

En este caso, el método de publicación de Route llama al método de lectura de FileController [4].



Laravel es el nombre de un framework creado para trabajar con PHP, creado en 2011 por Taylor Otwell y con el tiempo ha ido ganando terreno frente a otros frameworks que se especializan en trabajar con PHP, como Symfony o ZendFramework [4].

Es un framework joven con un gran potencial de futuro. Este cuenta con un equipo dinámico y poderoso que muestra archivos atractivos con contenido completo y fácil de entender; Además, proporciona las funciones necesarias para el desarrollo de aplicaciones comunes de manera fácil y segura. Resulta que tiene algunas cosas geniales como enrutamiento RESTful, cosas ligeras encantadoras o PHP nativo y otros extras. Basado en varios elementos de Symfony , Laravel también proporciona una base de código grande, segura y probada para sus aplicaciones web [3].

Laravel también proporciona un conjunto avanzado de herramientas para interactuar con la base de datos. Las migraciones de bases de datos le permiten diseñar y modificar fácilmente bases de datos independientes de la plataforma. Luego, la migración se puede ejecutar en cualquier tipo de base de datos compatible con Laravel (MySQL, PostgreSQL, MSSQL y SQLite) y no tendrá problemas de compatibilidad [6].

Características de laravel


El framework Laravel trabaja con una arquitectura de directorios avanzada, de manera que promueve la separación de archivos en un orden exacto y definido, lo que guía a todos los miembros del equipo y es un estándar en número proyectos diferentes. Por supuesto, también tiene una arquitectura en capas muy especializada que promueve la separación de códigos por responsabilidad. Su estilo arquitectónico es MVC. También contiene una amplia gama de funciones, que se utilizan para crear la mayoría de las aplicaciones web. Entre ellos se encuentran los siguientes:

.El sistema de enrutamiento facilita la creación y el mantenimiento de todos los tipos de motores de búsqueda y URL amigables, rutas API y más [3].

. Un sistema de abstracción de base de datos con un ORM poderoso pero fácil de usar que nos permite tratar los datos de la base de datos como objetos simples [3].

. El sistema crea colas de tareas para que pueda ejecutar tareas en segundo plano y aumentar el rendimiento de la aplicación [3].

. Múltiples configuraciones para enviar correos electrónicos con múltiples proveedores [3].

. Sistema de notificación de usuarios vía correo electrónico, base de datos [3].

. Una abstracción del sistema de archivos que nos permite escribir datos en los proveedores de la nube y, por supuesto, en los discos del servidor con el mismo código [3].

. Sistema de autenticación con todo lo que necesita, como solicitudes de contraseña, verificación de cuenta, solicitudes de inicio de sesión de usuario y más [3].

. La capacidad de acceder a los datos en tiempo real y recibir una notificación cuando cambie en la base de datos [3].

Historial de versiones


Laravel fue creado por Taylor Otwell en 2011, pero no fue hasta 2014 que comenzó a llamar la atención de los desarrolladores. De hecho, ese mismo año, el marco fue rediseñado [4].

Taylor decidió reescribirlo por completo, distribuir sus diversos componentes a través de Composer y reutilizar paquetes de terceros en el proyecto. Lo más destacado es el uso de componentes de Symfony, otro framework PHP muy conocido [4].

Esta reorganización fue una decisión inteligente. Mediante el uso de paquetes de terceros, no pierde tiempo creando lo que ya está en el mercado. Aun así, esta estrategia funciona bien con muchas tecnologías porque optimiza el trabajo de todos [4].

Con esto, es posible enfocarse en áreas más importantes, que acabaría dando a Laravel la identidad que tiene hoy [4].
