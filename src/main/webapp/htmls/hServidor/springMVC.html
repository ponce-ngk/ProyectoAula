Introducción al marco Spring Web MVC

El marco Spring Web Model View Controller (MVC) está diseñado en torno al envío de solicitudes de DispatcherServlet al controlador, con asignación de controlador configurable, resolución de vista, resolución de tema e idioma, así como soporte de carga de archivos de socket. . El controlador predeterminado se basa en las anotaciones @Controllery @RequestMapping, lo que proporciona una variedad de métodos de manejo flexibles. Con la llegada de Spring 3.0, el mecanismo @Controller también le permite crear aplicaciones y sitios web RESTful a través de la anotación @PathVariable y otras características [1].

En Spring Web MVC puede usar cualquier objeto como directiva u objeto de soporte de formulario. no necesita implementar una clase base o una interfaz específica del marco de trabajo. El enlace de datos de Spring es muy flexible: por ejemplo, trata los errores de desajuste de tipos como errores de validación que la aplicación puede evaluar, no como errores del sistema. Por lo tanto, no necesita copiar las propiedades de sus objetos comerciales como cadenas simples sin estilo en sus objetos de formulario solo para manejar envíos no válidos o para convertir cadenas correctamente. En su lugar, es mejor vincular directamente a sus objetos comerciales [1].

La resolución de pantalla de Spring es extremadamente flexible. El controlador suele ser responsable de preparar el modelo de mapa con los datos y elegir el nombre de la vista, pero también puede escribir directamente en el flujo de respuesta y completar la solicitud. La resolución de nombre de vista es altamente configurable a través de una extensión de archivo o acepta la negociación del tipo de contenido del encabezado, a través del nombre del bean, el archivo de propiedades o incluso una implementación personalizada de ViewResolver [1].

El modelo (la M en MVC) es una interfaz de mapa que permite una abstracción completa de la tecnología de visualización. Puede integrarse directamente con tecnologías de representación basadas en modelos, como JSP, Velocity y Freemarker, o generar directamente XML, JSON, Atom y muchos otros tipos de contenido [1].

El modelo Map simplemente se transforma en un formato apropiado, como atributos de solicitud JSP, un modelo de plantilla de Velocity [1].

. SWF le permite capturar flujos de páginas lógicas como módulos independientes que se pueden reutilizar en diferentes escenarios. Por lo tanto, es ideal para crear módulos de aplicaciones web que guíen a los usuarios a través de navegaciones controladas para impulsar los procesos comerciales [1].

Configuración básica

Lo común es que se configure la aplicación web de forma que todas las peticiones cuya URL sigue un determinado patrón pasen a través de Spring MVC. Así, por ejemplo, podemos "redirigir" a través de Spring MVC todas las peticiones que acaben en ".do".

Esto te permitira por ejemplo servir los recursos estáticos aparte, ya que no es necesario que estos pasen por el flujo de ejecución MVC.

Ejemplo

                            
                                 <servlet>
                                  <servlet-name> dispatcher </servlet-name>
                                  <servlet-class>
                                  org.springframework.web.servlet.DispatcherServlet
                                  </servlet-class>
                                  <load-on-startup>1 </load-on-startup> 
                                  </servlet>
                                  <servlet-mapping>
                                  <servlet-name> dispatcher </servlet-name>
                                  <url-pattern> *.do </url-pattern>
