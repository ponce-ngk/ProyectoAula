SPRING BOOT

¿QUÉ ES EXACTAMENTE SPRING BOOT?

Es un framework Java basado en Model View Controller, a través del cual, gracias a los componentes y librerías que proporciona, facilita el desarrollo e implementación de servicios REST .Eliminó la necesidad de configuración de aplicaciones mediante el uso de archivos XML, enfatizando el desarrollo de aplicaciones [1].

Dado que las principales características de Spring Boot que tenemos es que nos permite crear aplicaciones Spring independientes que tienen servidores de aplicaciones integrados como Tomcat, Jetty o Undertow, no es necesario crear un WAR para ejecutarlo, también permite la configuración automática de Bibliotecas Spring y de terceros, sin necesidad de configurar en archivos XML, lo que hace que la integración con otros proyectos Spring sea más fácil y rápida [1].

Spring Framework

Retrocedamos un poco y hablemos de Spring Framework. Con Spring Framework puedes crear aplicaciones empresariales de alto nivel, pero aún necesitas saber cómo funciona y especialmente cómo configurarlo. La configuración es uno de los elementos importantes del framework Spring. Puede separar las implementaciones personalizadas, las conexiones de bases de datos y las llamadas a servicios externos, lo que hace que Spring Framework sea más extensible, más fácil de mantener y de ejecutar. En algún momento, debe conocer las mejores prácticas para aplicar a una aplicación Spring. Comencemos con una aplicación Spring simple que demuestra cómo funciona Spring Framework [2].

Dado que las principales características de Spring Boot que tenemos es que nos permite crear aplicaciones Spring independientes que tienen servidores de aplicaciones integrados como Tomcat, Jetty o Undertow, no es necesario crear un WAR para ejecutarlo, también permite la configuración automática de Bibliotecas Spring y de terceros, sin necesidad de configurar en archivos XML, lo que hace que la integración con otros proyectos Spring sea más fácil y rápida [1].

Spring tiene varios frameworks complementarios que son:

. Spring MVC: este es utilizado para el desarrollo de aplicaciones web (este es parte e spring framework) [1].

• Spring Security: Este es utilizado para la inserción de funcionalidades para la autenticación y autorización [1].

• Spring Data: Destinado para aplicaciones que utilizan nuevas tecnologías de almacenamiento de datos como base de datos y servicios en la nube [1].

. Spring web Services: Este es utilizado para la inserción de funcionalidades para la autenticación y autorización [1].

• Spring web Flow: Este es una extensión de Spring MVC que permite la implementación de flujos (Wizards) entre pantallas [1].
