Introducción a las herramientas de desarrollo del lado del Servidor (Back-End)

¿Qué es el Back-End?

Un backend es un sistema comercial que se utiliza para administrar un sitio web o un negocio, como la administración de pedidos, el inventario y los sistemas de procesamiento de suministros. Este sistema recopila información de los usuarios u otros sistemas informáticos corporativos. es la responsable de gestionar la información que el usuario facilita a través del sitio web[1].

Un sistema de back-end es cualquier sistema que admita aplicaciones de back-office. Estos sistemas se utilizan como parte de la gestión social y funcionan tomando datos de las entradas de los usuarios en el sitio web y recopilando entradas de otros sistemas para proporcionar resultados receptivos[1].

Este término se utiliza para referirse al área lógica de cualquier página web. Nos referimos a la arquitectura interna del sitio que garantiza que todos los elementos realicen el correcto funcionamiento. No es visible para el usuario y no incluye ningún tipo de elemento gráfico[2].

Es la función de ingeniería que debe desarrollar el programador, ya que se basa únicamente en el código interno de la página. Esta área es responsable, además de la funcionalidad del sitio web, de la seguridad y optimización de los recursos[2].

Para algunos, el backend programa todo lo que el usuario final no ve, es decir, acceder a las bases de datos, procesar los datos ingresados por los usuarios y ejecutar un script, por ejemplo. Marina Rodrigo, profesora del curso online WordPress: diseñar y desarrollar páginas web desde cero, explica brevemente qué es el backend y explica brevemente sus diferencias con el frontend, otro concepto relacionado con el desarrollo web: “Casi todas las páginas web se dividen en dos partes. : el frontend (la parte que ve el usuario) y el backend (la parte trasera donde trabajamos y configuramos la web)"[3].

¿Qué funciones se gestionan desde el Back-End?

• Desde el Back-End se realizan todas las funciones que simplifican el proceso de desarrollo[2].

• Acciones lógicas [2].

• A partir del Back End, se utilizan bibliotecas de servidores web, tanto para comprimir imágenes web como para implementar el almacenamiento en caché u otros temas [2].

• La seguridad del sitio web también se mantiene [2].

• Gracias al Back-End se pueden optimizar los recursos para que las páginas sean más ligeras [2].

Además, hay una serie de ramificaciones que debe verificar para aprender a reprogramar:

1) Un buen desarrollador backend debe saber cómo configurar servidores. O al menos sabe lo suficiente como para hablar con el administrador de TI y explicarle sus necesidades[4].

2) Debe gestionar al menos un CMS, por ejemplo, WordPress [4].

3) Conocimiento de metodologías de programación [4].

4) Saber de memoria los conceptos básicos de programación y dominar al menos uno de ellos [4].

5) Ser MUY lógico y ordenado [4].
