Aplicaciones JAVA

¿Porqué se creo JAVA?

En 1991, el "Equipo Verde" dirigido por James Gosling de Sun Microsystems creó un lenguaje de programación para dispositivos digitales de consumo. El idioma se llamaba Oak, así que ¿por qué Oak? Porque había un roble frente a la oficina de Gosling[1].

El “Equipo Verde” demostró el uso del lenguaje con la televisión interactiva. Sin embargo, en ese momento era demasiado avanzado para la televisión digital por cable, y una tecnología que recién comenzaba a despegar, Internet, era más apropiada[1].

Más tarde, el idioma pasó a llamarse "verde" y finalmente a "Java" del café llamado "Java"; de ahí el logo de la taza de café.Debido a que C/C++ era popular en ese momento, James Gosling diseñó el lenguaje con una sintaxis de estilo C/C++ y la filosofía de "escribir una vez, ejecutar en todas partes". Años más tarde, Sun Microsystems lanzó la primera implementación pública de Java en 1995. Se anunció que el navegador de Internet Netscape Navigator incorporaría la tecnología Java. En 2010, Sun Microsystems fue adquirida en su totalidad por Oracle Corporation junto con Java[1].

JAVA SE

Java Platform, Standard Edition (Java SE) es una especificación que describe una plataforma Java abstracta. Proporciona una base para crear e implementar aplicaciones comerciales centradas en la red desde una PC de escritorio a un servidor de grupo de trabajo. Java SE se implementa mediante el kit de desarrollo de software (SDK) de Java[2].

Rule Execution Server puede ejecutar conjuntos de reglas con código 100% Java SE. Hay muchos casos de uso para ejecutar Java SE puro, como la ejecución por lotes o la ejecución de reglas desde un proveedor de Java Message Service (JMS) o Enterprise Service Bus (ESB), no JavaEE[2].

JAVA EE

Java Platform, Enterprise Edition (Java EE) se basa en la especificación Java SE. Representa una colaboración entre múltiples proveedores y líderes de la industria y proporciona soporte de infraestructura para aplicaciones[2].

Java EE es portátil y escalable y admite componentes heredados e integración basada en arquitectura EJB. Java EE simplifica las aplicaciones comerciales al definir y especificar un conjunto complejo de servicios estándar comunes, como nombres, administración de transacciones, concurrencia, seguridad y acceso a bases de datos[2].

En el marco Java EE, agregue las reglas en dos niveles:

• A nivel de aplicación, para gestionar la lógica empresarial dinámica y el flujo de actividad.

• En la capa de presentación, para personalizar el flujo de trabajo y el flujo de la página y para crear páginas personalizadas basadas en el estado de la sesión.

Aplicaciones JAVA

• Aplicaciones de Android: el lenguaje de programación Java utilizado por el SDK de Android (Kit de desarrollo de software) se usa generalmente para desarrollar aplicaciones de Android[1].

• Aplicaciones web: Java se utiliza para crear aplicaciones web utilizando servlets, Struts o JSP. Algunas de las aplicaciones web más populares escritas en Java son: Google.com, Facebook.com, eBay.com, LinkedIn.com, etc[1].

• Desarrollo de software: software como Eclipse, OpenOffice, Vuze, MATLAB, etc. utilizan Java[1].

• Procesamiento de Big Data: puede usar un marco de software popular como Hadoop (que a su vez está escrito en Java) para procesar Big Data. Para usar Hadoop, debe comprender la programación Java[1].

• Sistema de negociación: puede crear aplicaciones de negociación de baja latencia utilizando Oracle Extreme Java Trading Platform[1].

• Dispositivos integrados: mientras que los lenguajes de programación C/C++ siguen siendo opciones populares para trabajar con sistemas integrados, las tecnologías integradas de Java de Oracle proporcionan una plataforma y un tiempo de ejecución para miles de millones de dispositivos integrados como: televisores, tarjetas SIM, reproductores de Blu-disc. , etc[1].
