JavaServer Faces(JSF)

La tecnología JavaServer Faces simplifica la creación de interfaces de usuario para aplicaciones JavaServer. Los desarrolladores pueden crear aplicaciones web ensamblando componentes de interfaz de usuario reutilizables en una página; conectar estos componentes a una fuente de datos de aplicación; y conectar eventos generados por el cliente a controladores de eventos del lado del servidor. Este proyecto proporciona información sobre el desarrollo en curso de la especificación JavaServer Faces[1].

JavaServer Faces (JSF) es una tecnología JCP estándar para crear interfaces de usuario basadas en componentes en la plataforma Java EE. Este proyecto de GitHub en particular alberga el rastreador de problemas de especificación oficial de JSF. Hay tres implementaciones de la especificación JSF, todas desarrolladas bajo licencias de código abierto aprobadas por OSI[1].

Implementacion de JSF 2.3

JSF 2.3 y Java EE 8 están completos. Las implementaciones ejecutables de los hitos JSF 2.3, así como otras versiones, están disponibles en el repositorio javax.faces. Esta versión está incluida en GlssFish 5 Builds.

La especificación legible por humanos puede descargarse de http://jcp.org/.

La API está disponible en Maven Central en estas coordenadas.

                            <dependency>
                                <groupId>javax.faces</groupId>
                                <artifactId>javax.faces-api</artifactId>
                                <version>2.3</version>
                                <scope>provided</scope>
                            </dependency>

The implementation is also available at these coordinates.

                            <dependency>
                                <groupId>org.glassfish</groupId>
                                <artifactId>javax.faces</artifactId>
                                <version>2.3.8</version>
                                <scope>provided</scope>
                            </dependency>

Por lo general, las aplicaciones web que utilizan esta API/Marco incluyen la API JSF y los archivos Java Archive (JAR) de implementación en el archivo Web Application Archive (WAR). Esto no es necesario cuando estas aplicaciones web se implementan y ejecutan en WebSphere Application Server. Simplemente elimine estos archivos JAR junto con los archivos JSTL JAR del archivo WAR. Sin embargo, debido a que JavaServer Faces 2.2 es parte de Java Platform, Enterprise Edition (Java EE), una implementación de JavaServer Faces no se encapsula en una aplicación web cuando se ejecuta en un contenedor web compatible con la tecnología Java EE. JavaServer Faces se incluye con una aplicación web, se ignora porque la implementación de JavaServer Faces proporcionada por la plataforma siempre tiene prioridad[2].

El entorno de tiempo de ejecución de JSF para WebSphere Application Server no admite el uso de un único cargador de clases para toda la aplicación, ya que la inicialización de FacesConfig requiere un único cargador de clases para cada módulo JSF durante el inicio. Este soporte no está disponible cuando la aplicación contiene varios módulos web y uno de estos módulos es un módulo JSF. Por lo tanto, debe usar varios cargadores de clases cuando su aplicación contiene varios módulos web y al menos un módulo JSF[2].

Si una aplicación utiliza la implementación JSF proporcionada de MyFaces, no es necesario agregar org.apache.myfaces.StartupConfigureListener al archivo web.xml[2].
